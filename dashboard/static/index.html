How The Constructor Works

Use this page as the control center for full Android automation. Build a profile, tune the run, record manual paths, test, then start a safe run.

01

Run builder

Command combines the selected game, farm, methods, stages, credentials, prompts, and recordings into the environment used by main.py.

02

Game profiles

Profiles describe a game: package name, Play Store query, tutorial hints, purchase-preview hints, blockers, and gameplay strategy. Custom profiles are saved as JSON.

03

Presets

Presets save a complete run setup so a route can be reused or shared through MCP without re-entering every field.

04

CV autopilot

CV plans the next UI action from a screenshot. It can tap, type, swipe, press keys, wait, or stop when the goal is done.

05

Recordings

Manual taps, swipes, text, and keys can be recorded into repeatable paths and replayed for install, tutorial, or gameplay stages.

06

MCP

Any MCP-compatible AI client can inspect dashboard state, edit files, create profiles and presets, run checks, use CV, and control the connected Android device.

07

Vision Inspector

Inspect the latest screenshot with ROI, provider candidates, selected element, and latency. Draw a manual box to save templates and ROI zones.

08

Autopilot Builder

Write one goal prompt to generate GoalSpec, safety policy, screen graph, profile, ROI, templates, scenario, bundle, and validation reports.

09

Safety

Real purchases and final payment confirmations remain blocked. Google phone verification is manual and user-controlled.

10

Tests

Run Checks compiles active code, runs pytest, and enforces the 100% gate for deterministic MCP/CV/constructor modules.
